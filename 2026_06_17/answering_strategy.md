- After months of reading here and there on how best to tackle this, I have a flow here(not set in stone. mutable)

# Strategy

1. Gather requirements
2. List a few methods that system should do(in words) and tell interviewer that you will get to the entities based on this list
3. Come up with core entities and actors of the system based on list from step2
4. Write pseudocode of 2-3 most important flows(identified in step2)
5. Code up the flows from step4
6. Add design patterns/optimizations(can be done while in step 5 as well)

I have practised the parking lot system example using this exact framework
